- **Main.gs** — точка входа и основная логика выполнения.

  - function ExportLeadsToSheet — главный метод экспорта лидов:

  - function \_initializeConfig — сборка объекта конфигурации:
    инициализация рабочих листов, заголовков и временных периодов для
    работы скрипта.

  - function \_updateReferencesIfRequested — служебный метод:
    автоматическое создание и обновление справочников (источники, офисы,
    пользователи, поля лидов) при необходимости.

- **BitrixGetLeadsService.gs** — сервис для получения лидов из Bitrix.

  - function GetLeadsByFiltersMap — комбинированный сбор лидов:
    реализует логику «ИЛИ» (OR) для нескольких полей дат, выполняет
    дедупликацию записей через структуру Set и агрегирует данные в
    единый массив.

  - **Поддержка Batch-запросов** — инкапсулированная логика пакетного
    получения данных (по 50 команд) для ускорения работы с большими
    выборками.

  - **Оптимизация фильтрации** — последовательный перебор полей
    (например, «Дата создания» или «Дата квалификации») с визуальным
    логированием пересечений (Всего/Новых/Дублей).

- **SheetsService.gs** — работа с Google Таблицами (запись/чтение).

  - function BtxWriteDataToSheet — универсальный метод записи:
    оптимизированная выгрузка больших объемов данных (лидов/сделок) с
    разделением на этапы подготовки Rich Text, формирования массива
    строк и физической записи через setValues.

  - function PrepareLeadsSheet — подготовка листа: «жесткая» очистка
    содержимого и форматов, удаление лишних колонок, запись
    стилизованной «шапки» и закрепление первой строки.

  - **Оптимизация производительности** — внедрена система таймеров для
    контроля времени наложения ссылок (setRichText) и общей скорости
    записи данных.

  - **Автоматизация ссылок** — динамическое формирование кликабельных
    ссылок на карточки CRM (Лиды/Сделки) прямо в ячейках таблицы.

- **ReferenceSheetsService.gs** — работа со справочниками или
  вспомогательными листами.

  - function GetSourcesReference — выгрузка справочника источников:
    запрашивает данные из crm.status.list (тип SOURCE), выполняет
    трансформацию в DTO-формат и обновляет целевой лист со стилизацией
    шапки.

  - function GetOfficesReference — выгрузка элементов инфоблока

  - function GetFieldsInfo — сбор полной информации о полях Лида:

    (listLabel/formLabel) для корректного отображения русских названий.

  - function GetUsersReference — выгрузка пользователей: выполняет сбор
    данных о сотрудниках с поддержкой пагинации (while цикл), формирует
    полные имена и визуально помечает статус активности (✅/❌).

  - **Пакетная запись данных** — во всех методах используется
    принцип setValues(), что минимизирует количество обращений к Google
    Таблицам и ускоряет работу.

  - **Автоматизация UI** — функции включают автоматическую очистку
    листов, стилизацию шапок (разные цвета для разных сущностей),

  - function CallBitrix — универсальный вызов (REST 2.0): основной метод

  - function CallBitrix3 — вызов REST 3.0: метод для работы с новой

  - function ParseDateToIso — преобразование даты: приводит объект даты
    к строгому формату ISO 8601 (с учетом часового пояса Москвы +03:00)

  - function FormatUserName — форматирование имени: реализует

  - function GetReadableValue — универсальный конвертер (Lazy Loading):
    ключевой метод для превращения технических ID из API в читаемые

  - **Ленивая загрузка (Lazy Loading)** — справочники подгружаются в
    оперативную память только в момент первого обращения к конкретному

  - **Обработка сложных типов** — корректно обрабатывает множественные
    поля (через ;), денежные значения, флаги (превращая Y в ✅), а также
    телефоны и Email из структуры crm_multifield.

  - **Автоматическое приведение типов** — распознает и преобразует даты,
    числа и логические значения для корректного отображения в ячейках
    Google Таблиц.

## Шаблон моста для проекта таблицы

Ниже актуальный минимальный код для проекта таблицы, который использует библиотеку
`BitrixCore_Library`. Этот блок держим как «эталон», чтобы не копировать вручную из чатов.

```javascript
function onOpen() {
  BitrixCore_Library.initClient(SpreadsheetApp.getActiveSpreadsheet());
}

function runLeadExportManual() {
  BitrixCore_Library.ExportLeadsToSheet(SpreadsheetApp.getActiveSpreadsheet(), true);
}

// Запуск для триггера (тихий режим, без ручного UI)
function runLeadExport() {
  BitrixCore_Library.ExportLeadsToSheet(SpreadsheetApp.getActiveSpreadsheet(), false);
}

function runSetupTrigger() {
  const result = BitrixCore_Library.setupDailySync();
  SpreadsheetApp.getUi().alert(result);
}

// Legacy-метод (для старого Progress.html)
function getProcessingStatus() {
  try {
    return BitrixCore_Library.getLibraryStatus();
  } catch (e) {
    console.error("Bridge error (getProcessingStatus): " + e.message);
    return "❌ Ошибка моста";
  }
}

// Новый payload-метод (для обновленного Progress.html)
function getProcessingStatusPayload() {
  try {
    return BitrixCore_Library.getProcessingStatusPayload();
  } catch (e) {
    console.error("Bridge error (getProcessingStatusPayload): " + e.message);
    return { summary: "❌ Ошибка моста", log: "❌ Ошибка моста", done: true, canSaveDebugLog: false };
  }
}

// Кнопка «Сохранить отладочный лог»
function saveProcessingDebugLog() {
  return BitrixCore_Library.saveProcessingDebugLog();
}

// Очистка temp-лога при закрытии окна (best-effort)
function discardProcessingDebugLog() {
  return BitrixCore_Library.discardProcessingDebugLog();
}
```

### Именованные диапазоны для новых настроек

- `РазмерОкнаВыгрузкиЗвонков` — целое число часов (например, `72`).
- `ПодробныйЛогСинхронизации` — `TRUE/1/да` для включения temp debug-лога
  и кнопки ручного сохранения в конце синхронизации.
